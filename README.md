## 🤔 The Problem

Buying a used car in Sweden? You're gambling with your money.

- Dealers know everything about the car. **You don't.**
- Hidden problems? Recalls? Suspiciously low mileage? **You won't know until it's too late.**
- Existing tools give you raw data. **You need expert analysis.**

> **CarDeal fills the gap in the Nordic market** — we turn raw vehicle data into actionable intelligence.

## 👥 Who Is CarDeal For?

### 🧑 Private Buyers
> *"Is this car worth 300,000 SEK? Is it safe? Are there hidden problems?"*

Get instant answers — no mechanic visit needed before deciding.

### 🏪 Private Dealers (Småföretagare)
> *"I buy and sell 10 cars a month. I need to evaluate fast."*

Unlimited searches with our **Monthly** or **Lifetime** plan. Know the real value before bidding.

### 🏢 Corporation Dealers (Bilhandlare)
> *"Our sales team needs to evaluate inventory and incoming trade-ins."*

**Dealer Plan** — unlimited queries, API access, team accounts, bulk evaluation.

### 🔧 Workshops (Verkstäder)
> *"Customers ask us if their car is worth repairing or selling."*

Help your customers make informed decisions. Builds trust and repeat business.

### 🏦 Insurance Companies (Försäkringsbolag)
> *"We need risk profiles and accurate valuations for claims."*

API integration for automated vehicle risk scoring and market valuation.

### 🏬 4S Shops / Authorized Dealers
> *"We want to offer trade-in valuations backed by data."*

Premium API with your branding. White-label reports for your customers.

## 🏗️ Project Structure (For Developers)

```
cardeal/
├── src/                    # Rust backend (Axum + SQLx + Tokio)
│   ├── main.rs             # Application entry point
│   ├── config.rs           # Environment configuration
│   ├── models.rs           # Data models
│   ├── db.rs               # PostgreSQL + migrations
│   ├── cache.rs            # Redis cache layer
│   ├── repositories.rs     # Database queries
│   ├── services.rs         # Business logic
│   ├── ai.rs               # AI/LLM integration
│   └── handlers.rs         # API endpoint handlers
├── migrations/             # Database schema (PostgreSQL)
├── api/openapi.yaml        # API specification (Swagger/Postman)
├── frontend/
│   ├── android/            # Kotlin + Jetpack Compose
│   └── ios/                # Swift + SwiftUI
├── docs/                   # Multi-language documentation
├── Dockerfile              # Container build
└── docker-compose.yml      # One-command deployment
```

cp .env.example .env
docker compose up -d
curl http://localhost:8080/healthz
